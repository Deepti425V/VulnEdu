üóÑ Primary Data Sources

The foundational databases and organizations that provide the core vulnerability data used in VulnEdu.

National Vulnerability Database (NVD)

What: The U.S. government repository of vulnerability data based on CVE identifiers.
Why: Provides comprehensive CVSS scoring, detailed analysis, and machine-readable data feeds.
How we use it: Primary source for CVE details, severity scores, and real-time vulnerability data via API.
Official Website API Documentation Vulnerability API FAQ & Usage Guidelines

MITRE CVE Program

What: The authoritative source for CVE identifiers and the CVE naming standard.
Why: CVE IDs provide universal references for vulnerabilities across all security tools.
How we use it: CVE ID validation, cross-referencing, and understanding vulnerability disclosure processes.
CVE Main Site

MITRE Common Weakness Enumeration (CWE)

What: A comprehensive catalog of software and hardware weakness types.
Why: Provides classification framework for understanding root causes of vulnerabilities.
How we use it: CWE mapping, weakness analysis, and educational content about vulnerability patterns.
CWE Main Site CWE Data Downloads CWE Top 25

üîå APIs & Data Feeds

Technical interfaces and data formats used to programmatically access vulnerability information.

NVD API 2.0

RESTful API providing JSON responses for CVE data, CVSS metrics, and vulnerability details.
GET https://services.nvd.nist.gov/rest/json/cves/2.0
Rate Limits:
5 requests/30 sec (no key)
50 requests/30 sec (with key)
Response Format:
JSON with CVE objects
Key Parameters:
cveId, pubStartDate, lastModStartDate
